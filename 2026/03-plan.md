# 2026 年 3 月整体计划

> 月度定位：Q1 收口月。
> 核心任务不是扩张，而是把「漏洞挖掘主线 + 1 个 SaaS 验证点 + 内容复用链路」真正跑起来。

---

## 一、3 月总目标

### 1. 本月主题

- **Hunter 主导，Farmer 验证**
- 以漏洞挖掘提高收益上限
- 以 `kol.tools` 验证 SaaS 现金流可能性
- 以内容复用而非重度内容生产，完成品牌放大

### 2. 本月北极星指标

| 方向 | 3 月目标 | 验收标准 |
|------|----------|----------|
| 漏洞挖掘 | 完成 AI 高价值目标池筛选 | 至少筛出 `5` 个灾难级潜力目标，其中 `2` 个进入主攻 |
| SaaS | 完成 `kol.tools` 首个可上线 MVP | 至少上线 `1` 个工具页，可收集邮箱或直接付费 |
| 自媒体 | 完成 Claude Code 系列收尾与复用 | 完成系列复盘，拆出可复用素材并持续分发 |
| 现金流 | 建立 4 月可追踪的转化入口 | 至少形成 `1` 个可持续获客入口（SEO / 邮件 / 订阅） |

### 3. 资源分配

- 漏洞挖掘：`55%`
- SaaS：`30%`
- 自媒体/Newsletter：`15%`

> 3 月不再新增第 2 个 SaaS 开发项目。
> `BookFlow`、`51title`、`Ideas GPT` 继续保留，不进入本月主开发队列。

---

## 二、赛道计划

### 1. 漏洞挖掘

### 本月目标

- 完成 AI 目标池重排，严格按奖金天花板和结构性爆点筛选
- 固定 `2` 个主攻项目、`3` 个跟进项目
- 至少推动 `1` 个项目进入可提交或可深度沟通状态

### 关键动作

- 建立 3 月攻击池清单：只保留奖金预期 `>= 50,000` 的 AI 目标
- 每个目标必须写清一条“为什么结构上一定会出问题”的假设
- 对主攻目标输出攻击面地图：多租户、Agent 权限、工具调用、上下文隔离、沙箱逃逸
- 对已验证无大机会的目标快速止损，不做低危堆量

### 月底验收

- [ ] 已筛出 `5` 个高潜 AI 项目
- [ ] 已明确 `2` 个主攻项目与 `3` 个跟进项目
- [ ] 已形成至少 `1` 个高价值漏洞叙事或提交材料
- [ ] 没有因为“保持忙碌”而转去做低赔率目标

### 2. SaaS

### 本月聚焦

- 唯一执行项目：`kol.tools`
- 唯一目标：上线第一个足够快、足够轻、足够可传播的工具 MVP

### 本月交付

- `YouTube Title Generator` 作为第一工具优先上线
- 完成落地页、工具页、基础额度策略、转化入口
- 如果付费链路来不及，至少先完成：
  - 邮箱收集
  - 候补名单
  - 使用次数限制
  - 基础埋点

### 成功标准

- [ ] MVP 可公开访问
- [ ] 至少有 `1` 个可量化转化入口
- [ ] 至少完成 `10` 次真实用户使用或种子用户反馈
- [ ] 明确 4 月是否继续加码的判断依据

### 暂缓事项

- 不在 3 月并行开发 `BookFlow`
- 不启动 `51title`
- 不扩写大而全工具矩阵，先做单工具验证

### 3. 自媒体 / 品牌

### 本月定位

- 不开新大坑
- 以复用已有 Claude Code 系列素材为主
- 自媒体服务于 SaaS 获客与个人品牌，而不是挤占主线时间

### 本月动作

- 完成 Claude Code 系列发布收尾、数据复盘、选题总结
- 每周从现有长视频中拆出短内容，分发到 `X / YouTube Shorts / 小红书`
- 为 `kol.tools` 准备 1 组演示型内容
- AIPwn 保持轻量更新，重点是不中断 Newsletter 与账号活跃度

### 月底验收

- [ ] 完成 Claude Code 系列 3 月复盘
- [ ] 至少产出 `8-12` 条拆条内容
- [ ] 至少发布 `1` 条与 `kol.tools` 直接相关的引流内容
- [ ] AIPwn Newsletter / X 保持基本活跃

## 三、周节奏

### 第 1 周（2026-03-01 至 2026-03-08）

- 完成 3 月攻击池清单与目标排序
- 锁定 `kol.tools` 的 MVP 范围，只保留第一工具
- 清理 2 月遗留内容任务，停止无关并行事项
- 输出 Claude Code 系列复盘提纲

### 第 2 周（2026-03-09 至 2026-03-15）

- 漏洞挖掘进入双主攻项目深挖
- `kol.tools` 完成产品页与核心生成流程
- 建立基础埋点、收集入口和反馈渠道
- 发布首批拆条内容

### 第 3 周（2026-03-16 至 2026-03-22）

- 优先推动漏洞线索成型，准备提交或沟通材料
- `kol.tools` 内测，获取第一批种子反馈
- 根据反馈压缩功能，不做无关打磨
- 发布 1 条工具演示内容

### 第 4 周（2026-03-23 至 2026-03-31）

- 对漏洞主线做阶段复盘，判断目标池是否需要重排
- `kol.tools` 对外上线或小范围公开
- 汇总 3 月数据：目标数、进展、访问、反馈、订阅
- 形成 2026 年 4 月计划输入

## 四、3 月不做什么

- 不同时推进多个 SaaS 开发项目
- 不为了内容活跃度牺牲漏洞主线时间
- 不做无法指向现金流或高赔率命中的忙碌型任务
- 不因为短期无反馈就频繁换赛道

---

## 五、月底复盘问题

到 `2026-03-31`，只回答下面 5 个问题：

1. 是否已经找到 `5` 个真正值得重注的 AI 目标？
2. 是否有至少 `1` 个漏洞机会进入高价值阶段？
3. `kol.tools` 是否已经得到真实用户反馈，而不是自我感觉良好？
4. 内容是否形成了对产品或品牌的真实放大，而不是单纯消耗时间？
5. 4 月应该加码的唯一方向是什么？

---

## 六、3 月结论

3 月不是“多做一点”的月份，而是“砍掉杂音、验证主线”的月份。

如果本月结束后，漏洞目标池仍然模糊、`kol.tools` 仍未上线、内容仍无法复用，那么 Q1 其实没有闭环，4 月必须继续收缩，而不是扩张。
