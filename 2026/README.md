## 监控与复盘 (Rhythm & Metrics)

### 1. 仪表盘 (Passive & Real-time)
> **原则**：只看结果，不看过程。不要人工记账。
*   **资金**：Stripe Dashboard / 银行余额 (Cash In)
*   **流量**：Google Search Console / YouTube Studio
*   **自媒体**：粉丝数
*   👉 **[查看自动化仪表盘技术方案](./docs/DASHBOARD_SPEC.md)** (GitHub Actions + Telegram)

### 2. 复盘节奏 (Check-in)
*   👉 **[使用复盘模版 (周/月/季/年)](./docs/REVIEW_TEMPLATES.md)**
*   **周冲刺 (Weekly)**：
    *   **Hunter**：本周挖掘漏洞数 / 提交报告数 (个)
    *   **Farmer**：本周新增MRR / 活跃用户数 (人)
    *   **Action**：周日晚 15 分钟，记录本周**总到账金额**。
*   **月度复盘**：对照季度目标与当月现金流，判断是否需要加速/止损。
*   **季度复盘**：围绕执行节奏表进行偏差分析，必要时调整项目组合。
*   **年度复盘**：评估是否达成结余目标，确定下一年主题与资源分配。
