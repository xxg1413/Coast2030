## 三、aipwn.org 架构规划

### 3.1 域名架构

| 域名 | 用途 | 托管方案 |
| :--- | :--- | :--- |
| **aipwn.org** | Newsletter + 博客 (主站) | Substack 托管 |
| **app.aipwn.org** | SaaS 产品 (未来) | Vercel |
| **tools.aipwn.org** | 开源工具集 (未来) | Vercel / GitHub Pages |

> **原则**：主站零运维，专注内容产出；产品用二级域名独立部署

### 3.2 Substack 主站内容

**首页 About**:
```
Hunt AI. Hack Future.

我是 pxiaoer，专注 AI 产品安全研究。
每周分享最前沿的 AI 安全洞察。

🏆 Hall of Fame: [漏洞荣誉榜]
```

**内容分类 (Sections)**:
- 📬 **Weekly** - 每周 AI 安全速递
- 🔍 **Deep Dive** - 深度案例分析
- 🏆 **Hall of Fame** - 漏洞荣誉榜 (置顶页面)
- 💡 **Insights** - 判断方法 / 思考碎片

### 3.3 技术选型

| 组件 | 方案 | 理由 |
| :--- | :--- | :--- |
| **主站** | Substack | 零运维，自带订阅/付费系统 |

| **支付 (产品)** | Stripe | 全球付费 |

## 六、与漏洞挖掘的协同

> **核心原则**：内容为漏洞服务，而不是反过来

### 6.1 协同方式

| 漏洞挖掘阶段 | 内容输出 |
| :--- | :--- |
| **研究中** | 不输出细节，只分享判断方法 |
| **已提交，待修复** | 完全保密 |
| **已公开** | 深度复盘 + Newsletter 专题 |
| **谈判中** | 适度展示专业度，提高议价权 |

### 6.2 时间分配

| 活动 | 每周时间 | 说明 |
| :--- | :--- | :--- |
| 漏洞挖掘 | 30-40 小时 | 核心产出来源 |
| Newsletter 写作 | 3-5 小时 | 周五发布 |
| X 运营 | 1-2 小时 | 碎片时间 |
| 网站维护 | 1 小时 | 可批量处理 |
