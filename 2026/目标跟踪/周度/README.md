# 周度计划与复盘指南

> 简洁高效的周度工作流

---

## 🎯 新工作流（2分钟完成）

### 周日晚上 5 分钟

1. **打开 `2026-tracking.md`**
2. **填写下周关键事件**（在对应行写 3-5 个字）
3. **更新 metrics 文件**（2分钟填数字）
4. **完成**

---

## ✍️ 写法说明

### Level 1: 极简版（推荐日常使用）

在 `2026-tracking.md` 对应行的 `#` 列写简短关键词：

| 列 | 写什么 | 示例 |
|----|--------|------|
| 💰 现金 | 到账金额 | `¥15,000` |
| 🐛 Hunter | 关键进展 | `首单$2k` |
| 🚀 SaaS | 关键事件 | `🚀发布` |
| 📱 自媒体 | 关键数据 | `X+500` |
| 复盘 | 链接（需要时） | `#` |

**示例行：**
```markdown
| W06 | 02/03-02/09 | - | - | 🚀2/14发布 | X+200 | # |
```

---

### Level 2: 标准版（有重要事件时）

在表格后添加简要说明：

```markdown
| W06 | 02/03-02/09 | - | - | 🚀2/14发布 | X+200 | # |

**W06 关键事件：**
- 2/14: Wukong CLI v1.0 发布
- GitHub Stars: 500+
- 发现 2 个漏洞线索
```

---

### Level 3: 详细版（需要深度复盘时）

创建单独复盘文件 `周度/W06-复盘.md`：

```markdown
# W06 复盘 (02/03-02/09)

## 数据
- 现金: -
- Hunter: 0提交
- SaaS: 🚀发布
- 自媒体: X+200粉

## Big 3 回顾
1. Wukong发布 ✅
2. 首单 ❌ 推迟
3. 涨粉 🟡

## 下周调整
- 修复Bug优先
- 漏洞挖掘每天2h
```

然后在 `2026-tracking.md` 的复盘列链接到该文件：
```markdown
| W06 | ... | ... | ... | ... | [复盘](./周度/W06-复盘.md) |
```

---

## 📋 周度工作检查清单

### 每周日必做（5分钟）

- [ ] 填写 `2026-tracking.md` 本周行
- [ ] 更新 `metrics/cash-flow.md`（如有到账）
- [ ] 更新 `metrics/social-stats.md`（粉丝数）
- [ ] 确定下周 Big 3（心里清楚即可）

### 每月末必做（30分钟）

- [ ] 回顾本月所有周数据
- [ ] 完成月度复盘
- [ ] 调整下月计划

---

## 💡 建议

| 场景 | 建议做法 |
|------|----------|
| 普通周 | 只在 tracking.md 表格填数字 |
| 发布/里程碑周 | 表格 + 简要文字说明 |
| 失败/重大决策周 | 创建详细复盘文件 |
| 月底 | 汇总到月度复盘 |

---

**核心原则：5分钟能完成的，绝不用10分钟**
