# Coast 2030 · 个人长期计划总纲

> 核心原则：不押未来奇迹，只追踪当下可兑现的现金流与可复用能力。

## 终极目标（2030）

- 状态：可自由退休，不依赖单一平台或单一产品
- 资产：现金结余 > `500 万`（理想 `2000 万`）
- 系统：拥有可持续运转的个人事业系统

## 年度路线图

| 年份 | 总收入目标 | 结余目标 | 关键主题 |
| :--- | :--- | :--- | :--- |
| 2026 | 300 万 | 100 万 | 现金流验证 |
| 2027 | 400 万 | 150 万 | 业务扩张 |
| 2028 | 800 万 | 500 万 | 爆发增长 |
| 2029 | 1500 万 | 1000 万 | 稳定收割 |
| 2030 | 2500 万 | 2000 万 | 自由兑现 |

## 执行策略

| 维度 | Hunter（投机路径） | Farmer（投资路径） |
| :--- | :--- | :--- |
| 方向 | 漏洞挖掘 / 热点工具 | SaaS 主线 / 个人 IP |
| 关键词 | 快速验证、高回报、高止损 | 复利、留存、护城河 |
| 目标 | 抬高收益上限、补充现金流 | 构建稳定底座与长期增长 |

## 复盘节奏

- 月度复盘：对照年度目标与当月现金流，决定加速/止损
- 季度复盘：检查项目组合、投入产出与主线偏移
- 年度复盘：评估结余目标达成情况，重置下一年主题

## 使用方式（当前）

1. 日常记录与追踪：使用 Dashboard（任务、收入、进度）
2. 长文档与项目资料：维护在 `2026/` 目录
3. 如文档与系统数据不一致：以 Dashboard 数据为准

## 铁律

1. SaaS 上线后若连续 3 个月未达到关键付费指标，直接止损
2. 同时在研项目不超过 2 个
3. 现金为王：只相信已到账收入
4. 复盘优先于扩张：先闭环，再加速
